## WHAT WE BUILT

**AdversarialArena** -- an OpenEnv-compatible RL environment where two AI agents
battle in real time:

```
ATTACKER (Gemma 3 1B + GRPO)       DEFENDER (Gemma 3 4B + Online Fine-tune)
  Learns to generate injections  <-->  Learns to catch injections
  that evade the defender              by training on what it misses
         |                                        |
         +-------------- ARMS RACE ---------------+
                   Both get stronger.
```

### The Environment Structure

```python
env = ArmsRaceEnv(
    episode_rounds=8,    # rounds per episode
    group_size=8,        # GRPO sample size
    curriculum=True,     # difficulty scales with defender strength
)

obs    = env.reset()
result = env.step_round(attacks, caught_flags)
stats  = env.global_stats()
```

Fully Gymnasium-compatible. Wraps into OpenEnv's step / reset / state API.

## HOW IT WORKS

### Round loop (every ~2 minutes on A100):

```
1. ATTACKER generates 8 attack variants via GRPO sampling
   - Temperature scales with difficulty (more creative as it gets harder)
   - 8 attack types: direct override, roleplay jailbreak, authority claim...

2. DEFENDER classifies all 8 (Gemma 3 4B, 4-bit quantized, JSON output)
   - Returns: injection/benign + severity + attack vector

3. REWARDS computed:
   - Attacker: +1 per evasion, -1 per catch
   - Group Relative Policy Optimization:
     advantage = (reward - group_mean) / (group_std + eps)
     loss = -mean(advantage * log_prob) + 0.001 * entropy_bonus

4. ATTACKER updates via GRPO (policy gradient, AdamW, grad clip 1.0)

5. DEFENDER updates every 3 rounds from replay buffer
   - Over-samples evasions (hard negatives)
   - Online fine-tune: 3 gradient steps, batch=8

6. CURRICULUM advances difficulty if defender detection rate > 80%
   - Easy:   direct_override, system_override only
   - Medium: + roleplay_jailbreak, postscript_injection
   - Hard:   all 8 attack types unlocked
```

### Why GRPO specifically

Standard policy gradient needs a value network. GRPO doesn't --
it uses the group itself as the baseline.
Sample 8 attacks. Normalize rewards within the group.
Perfect for adversarial generation where reward is binary and sparse.

```python
rewards   = torch.tensor(rewards).reshape(K, G)
advantages = (rewards - rewards.mean(dim=1, keepdim=True)) / \
             (rewards.std(dim=1, keepdim=True) + 1e-8)
```

Same algorithm that trained DeepSeek-R1.
We applied it to adversarial text generation.

1. Systematic bug diagnosis -- truncation at 800 chars silently killed hard tasks
